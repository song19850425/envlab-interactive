WorkBuddy 从 0 到 1：一份让妈妈也能用的 AI 助手教程

原创 小宋的环保笔记 2026-07-03

之前发布了《Windows/Mac 安装 Claude Code / Codex》之后，确实帮助了很多小白使用上了 AI 工具。

不过毕竟是海外工具，无论是注册、支付、安装都还是有门槛，还是有人用不上。

而工作还得继续。汇报要写、数据要理、PPT 要出、代码要改。

所以决定再补一篇关于国产 Agent —— WorkBuddy 的教程。

坦白说：WorkBuddy 目前是我在国产通用 Agent 里，愿意持续打开的少数几个之一。

如果你不打算硬啃开发流程，也不想折腾环境，那可以说：日常办公，七成以上的活，它都能接。

我拿一个"办公场景"加一个"开发场景"两条线，把整个流程串一遍。跟着走一遍，你就上手了。

⬇️

一、下载与安装

第一步，装上它。

官网直接进：workbuddy.cn

Mac、Windows 两个版本都有，适配自己系统选择下载。安装包下下来，双击一路默认，装完打开客户端。

登录环节是本篇第一个"友好点"：直接微信扫码。

说真的，作为一个平时被各种 API Key、双重验证、代理配置折磨麻木的人，第一次看到微信登录框弹出来的时候，愣了两秒才反应过来——原来 Agent 产品也可以不折腾。

扫码 → 授权 → 进主界面。

二、主界面

第一次进去，先别急着发消息，花两分钟熟悉一下布局，后面用得顺不顺，全看这一步。

中间：对话区 + 三种场景

右侧那块大对话面板，就是你和 WorkBuddy 打交道的主战场。上方能切三个"场景"：

🎯 三个工作场景
💻 代码开发 📋 日常办公 🎨 设计创意

这三个场景不是摆设。你切到哪一个，下方的「快捷入口」就跟着变。

切到「日常办公」，变成文档处理、数据分析可视化、深度研究、幻灯片。

切到「代码开发」，底下摆的是日常开发、网站开发、Agent 应用、Skill 开发。

切到「设计创意」，就是网站设计、App 端、PPT、原型、品牌。

每个入口再往里点，还有一层。以「日常办公 → 文档处理」为例，你会拿到一叠预置的提示词模板，你只要把里面的字段换成自己的实际需求，改两下就能直接发。这一层"模板化"对新手极友好，可以说是国内产品里做得比较用心的。

对话框下方左侧的这一排

从左到右过一遍。

1. 工作模式（三档）

  • Craft：完整的 Agent 模式，接活就直接干，日常主要用它。
  • Ask：只聊天，问一句答一句，等价于豆包这种普通对话。
  • Plan：先给你出计划，你点头它再动手。新手怕 AI 乱开火，先用它过一遍最稳。
⚠️ 一个坑 WorkBuddy 默认是 Craft 模式。也就是你随便发条消息，它就直接开始跑了，积分同步开始扣。

所以只是想问句话，先切 Ask；还没想清楚怎么办，先切 Plan。

2. 模型选择

WorkBuddy 把国产主流模型都塞进来了：腾讯混元、DeepSeek、GLM、Kimi 这些。不同模型消耗的积分不一样。

选哪个？推荐 DeepSeek V4 Pro：价格友好、综合能力扎实，世界知识、编码、Agent 能力几乎都在合格线以上，除了缺多模态，其他基本是水桶型。

想接自己的 API Key 也可以，走「设置 → 模型 → 添加模型」，兼容 OpenAI 协议的接口都吃。

3. 权限

两档：默认权限、完全访问。我自己的习惯是选完全访问——那些弹窗提示我大部分也看不懂，最后都是无脑点确认，不如省心一点让它自己跑。

4. 对话记忆

打开后它会从聊天里提炼关键信息存起来，下次遇到相关话题自动调用。用得越久，它对你的偏好、语气、习惯就越有把握。

右侧的加号 & 工作空间

点开是上传入口。本地文件、腾讯文档、知识库都能塞进来。特别提一句：它跟 ima 知识库全面打通。

「工作空间」你把它当成"项目文件夹"就行。任务开始前挑一个目录，WorkBuddy 会基于这个目录读文件、写产物。

💡 一句真心话：前期把分类做好。前面偷一下懒，后面找东西真的能崩溃。

三、专家、技能与连接器

从左侧点「专家」进去，你会看到顶部有三个 tab：专家 / 技能 / 连接器。

三者不一样，分开看。

专家：封装好的垂直 Agent

专家 = 已经调好人设、装好工具、配好知识的一个垂直方向的 Agent。

WorkBuddy 内置了 100+ 个专家，按行业分类摆好。每个专家一张卡片，写清楚它擅长啥。点进去就是对话，用法和普通对话一模一样，区别是背后已经换了一个"带专业视角"的 Agent。

举几个例子：

  • 内容创作专家：擅长创作引人入胜的多平台内容
  • UI设计师：精通设计系统和组件库，追求像素级完美
  • AI图像提示词工程师：精通AI图像生成的语言密码

专家团：多 Agent 协作

专家是一个 Agent 干活，专家团就是一群 Agent 分工干活。你把需求扔给专家团，团里的"负责人"会自动拆任务、分派、并行推进，最后把结果给你收拢起来。

代价是：积分消耗大概是单专家的 3–5 倍。有预算再上。

技能（Skills）

技能这个概念其实不新，很多国外 Agent 都有。但 WorkBuddy 做了一个像样的 Skills 市场，这一步对国内用户来说非常关键——不用自己到处找、到处装。

进去分几个区：推荐官方精选、Skillub 社区作品、套件（插件包）。想自己做一个？右上角「添加技能 → 创建技能」，一句大白话讲需求。

连接器（MCP）

连接器就是 MCP，把 WorkBuddy 和你日常在用的外部服务打通。

WorkBuddy 在这块最大的优势，是无缝接腾讯自家生态：

  • QQ 邮箱：直接收发、搜索、整理邮件
  • 腾讯会议、腾讯文档：都能接
  • 通达信 MCP：看行情，炒股党狂喜

配置极简单：扫码 → 手机端授权 → 完事。其他服务都是这个套路，跟着页面提示走就行。

四、开工前把设置调好

点右上角头像 → 打开「设置」。

系统设置里的两个开关

  • 技能自动更新 —— 开。安装的技能会跟着升级，一劳永逸。
  • 锁屏远程 —— 开。电脑不进休眠，出去开会任务照跑。

个性化：自定义指令

这项相当于给整个 WorkBuddy 定"总纲"。你在别处写过 CLAUDE.md 的话，理解成一回事。

下面是一份很好用的模板，直接抄走：

# 减少常见 LLM 编码错误的行为准则

## 1. 先想清楚再写
- 先理解任务目标再开始修改代码
- 关键假设必须说明
- 如果有多种理解，先列出可能解释

## 2. 简洁优先
- 用最少、最直接的代码解决当前问题
- 不添加未要求的功能

## 3. 精准修改
- 只修改必需的代码
- 不顺手重构、不顺手优化

## 4. 目标驱动
- 多步任务先给简短计划
- 修改后说明改了什么、如何验证

## 5. 验证优先
- 优先用现有测试方式验证
- 修复 bug 时先解释根因

五、案例一：办公任务

先来个办公向的实战。WorkBuddy 在办公场景下确实下了不少功夫。

流程：

  1. 切到「日常办公」场景
  2. 选一个工作空间目录，不选也行，对话框右边加号临时上传
  3. 挑一个模型
  4. 输入需求

发消息旁边有个 「优化提示词」 按钮。新手强烈推荐点一下，它会自动把你那句大白话，改造成一份结构化的提示词。

💡 小技巧：对国产模型来说，喂结构化提示词的效果差异很明显——这是有一堆研究撑腰的结论，不是我瞎讲。

跑完之后：右侧栏能看到产物文件和预览。也可以点开始工作空间目录，直接看到本地生成的东西。

六、案例二：开发任务

再来一个开发向的案例，vibe coding 一个小网页。

流程：

  1. 切到「代码开发」场景
  2. 点「网站开发」
  3. 用 Plan 模式，先出方案
  4. 点「优化提示词」，回答反问
  5. 确认，让它自己跑

然后还可以进一步，挂一个自己写的 Skill 继续跟 WorkBuddy 说："用我那个服务器 Skill 把这个网页部署上线。" 它会照办。

🤔 关于开发这块讲句实话：
国产模型在代码上离最顶尖的那批还有差距。小项目、小网页、小工具没问题，但真上复杂大工程，我依然强烈建议你想办法用上 Claude Code 或 Codex。这个话搁两年后再看会不会翻，我也不知道，但眼下就是这样。

七、远程控制：不在电脑前，也能让它干活

这是我个人非常喜欢的一块。因为是腾讯自家产品，WorkBuddy 的远程控制原生打通了微信——这一点很多海外 Agent 想做都做不到。

配置步骤：

  1. 打开左侧功能区里的「助理」
  2. 点「设置」，把开关打开
  3. 掏出手机，微信扫码，进入小程序授权登录
  4. 客户端左上角出现"已连接"字样，就 OK

手机端小程序里有两种模式：

  • 云端工作：任务在云端沙箱跑，电脑关着都行
  • 连接电脑：远程操控你本地这台机器，适合需要读本地文件的任务
举个真实场景：我人在外面，突然被催"把电脑里某份文件发给财务"。搁以前，只能赶回去处理。现在我掏出手机，在小程序里跟它说一声就完事。

写在最后

我一直有个观点：对绝大多数普通人来说，Agent 最大的价值不是帮你写出漂亮代码，而是把每天那些琐碎、重复、让人烦躁的办公任务自动化掉。

发一封邮件、整理一批数据、出一份 PPT、跑一个小站——这些活加起来，才是"AI 真正让生活变轻"的部分。

WorkBuddy 现在把这几件事都接住了，而且做得挺体面。

之前写 Claude Code 和 Codex 教程时，最扎心的评论永远是那句「好是好，用不了」。看了几百遍还是无奈。这一篇算是把那块缺的拼图补上了。

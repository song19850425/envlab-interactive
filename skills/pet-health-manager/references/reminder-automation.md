# 逾期主动提醒 · automation 设计

纯前端 Demo 的提醒是**被动**的（打开页面才显示）。要变成**主动推送**，用 WorkBuddy automation 每周扫描。

## automation 配置要点
- 名称：宠物健康周提醒
- 周期：`FREQ=WEEKLY;BYDAY=SU;BYHOUR=20` （每周日 20:00）
- 工作目录：放 Demo HTML 的项目根

## 执行逻辑（prompt 草稿）
```
1. 读取宠物记录数据源（JSON 或让用户导出）
   - 若用户把记录存在某个 .json，直接读取
   - 否则提示用户：在 Demo 页面导出当前数据（localStorage→文件）
2. 对每只宠物扫描：
   - 驱虫：days(last.date, today) > last.cycle → 逾期 N 天
   - 疫苗：last.next < today → 已到期
3. 生成"本周待办清单"Markdown：
   | 宠物 | 类型 | 状态 | 建议 |
4. 输出清单给用户（用户再转发给对应主人）
5. 可选：接入微信模板消息/短信后，改为自动发送
```

## 升级路径
| 阶段 | 能力 | 需要 |
|------|------|------|
| L1 被动 | 打开页面显示逾期 | 仅 Demo，已具备 |
| L2 周报 | 每周生成清单给你 | WorkBuddy automation（无需额外账号） |
| L3 主动 | 自动推送到主人手机 | 微信模板消息/短信 API + 用户授权 |

## 注意
- MVP 验证期做 L1+L2 即可，L3 等真实用户量起来再做
- 提醒文案保持"工科爸爸"人设：数据 + 不吓人 + 给行动建议
- 不要伪造"已发送"——L2 阶段明确告诉用户"这是待转发清单"

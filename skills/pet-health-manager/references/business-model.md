# 商业化路径（宠物健康管家）

## 市场判断
- 中国宠物主超 1 亿，但现有 App（波奇/宠物家）只做"记账"或"社区"
- 核心未满足点：**没有 App 把"记录→提醒→复购"做成无脑顺手的习惯闭环（Track-on-Time）**
- 突破口 = "超时未驱虫→自动催" 强制钩子 → 留存 → 复购入口

## 与已有资产咬合
| 已有资产 | 在本项目用法 |
|---------|------------|
| 交互式 HTML（EnvLab 同款） | Demo 即 MVP，不用上架 App，扫码即用 |
| loop-engineering 自动化 | "超时驱虫→自动提醒" = 每周 automation |
| 工科爸爸人设 + 小红书 | 宠物健康科普引流，复用内容管线 |

## 变现由轻到重
```
L1 佣金：驱虫药/疫苗 CPS 分润（零库存，ARPU ¥20-60/年）
L2 会员：多宠管理 + 报告解读 + 问诊入口（¥99/年）
L3 B端：宠物医院 SaaS 引流工具（按门店收费）
```

## 最小验证动作（本周可做）
1. 发小红书/朋友圈：标题「给猫主子做了个健康管家，超时驱虫自动催我」
2. 用 Demo 截图 + 甩可分享链接
3. 收 50 个真实宠物主反馈 → 判断是否值得做 App
4. 提醒自动化用现有 automation 能力跑通，几乎零边际成本

## 局限（MVP 阶段可接受）
- 换手机不同步（localStorage 跟浏览器走）→ 真用户提了再加云端
- 提醒被动（打开才显示）→ 用 automation 升 L2

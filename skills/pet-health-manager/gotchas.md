# 宠物健康管家 · gotchas（失败案例）

> 这些坑在首版 Demo 和同类"记录+提醒"工具中都踩过或极易踩。

1. **别做后端/数据库**：MVP 阶段用 `localStorage` 即可。一旦上后端就要管账号/隐私/合规，验证期完全没必要。换手机不同步是已知局限，等真实用户提"丢了"再做云端。

2. **iOS 输入框缩放回弹**：`input/select` 的 `font-size` 必须 ≥16px，否则聚焦时整个页面放大。已在中强制 16px。

3. **体重曲线在手机上糊**：必须用 `devicePixelRatio` 重设 canvas 分辨率（`cv.width=cssW*dpr` + `ctx.setTransform(dpr,...)`），否则 Retina 屏线条发虚。

4. **逾期计算用本地日期**：`days()` 用 `new Date(b)-new Date(a)` 毫秒差除 86400000，注意时区——存日期用 `YYYY-MM-DD` 字符串，`todayStr()` 也返回该格式，避免 UTC 偏移导致差一天。

5. **首次打开必须有内容**：`load()` 里 `if(!pets.length)seed()`，预置 1-2 只示例宠物（一只正常、一只逾期），否则空页面用户秒关。

6. **提醒只能被动展示，别假装能主动推送**：纯前端 Demo 的"提醒"是用户打开页面才显示。要主动催，必须配 automation/微信模板/短信——在 Demo 阶段明确告知用户这一边界，别过度承诺。

7. **标签栏要吸顶**：`.tabs{position:sticky;top:0}` 否则长表格下滑后切换标签要滚回顶部，手机体验差。

8. **数据模型用"数组+当前选中"而非多个全局对象**：多宠物场景下 `pets[]` + `cur=pets[i]` 比 `pet1/pet2` 全局变量好维护，增删宠物不用改结构。

9. **别用纯黑纯白背景**：用户偏好赛博朋克暗色，但用 `#0e1117` 而非 `#000`，文字用 `#e6edf3` 而非 `#fff`，降低刺眼感。

10. **"添加到主屏幕"提示要显眼**：很多用户不知道网页能加主屏。Hero 区放一行引导文案，是"像 App"体验的关键一步。
